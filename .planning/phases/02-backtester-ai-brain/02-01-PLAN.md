  <action>
    Write tests first (RED), then implement src/backtester.py (GREEN).

    src/backtester.py must contain:

    1. `BacktestEngine` class:
       - `__init__(self, log_file: str, last_n: Optional[int] = None)` — stores path and limit
       - `load_trades(self) -> list[dict]` — reads JSONL file line by line, parses each with json.loads, wraps in try/except to skip malformed lines (log warning). If last_n is set, return only the last N entries. If file doesn't exist, return empty list and log warning.

    2. `BacktestReport` dataclass:
       - Fields (all Decimal): win_rate, total_opportunities, avg_profit, max_profit, max_loss, profitable_count, losing_count
       - `profit_buckets: dict[str, int]` — distribution buckets

    3. `compute_report(trades: list[dict]) -> BacktestReport`:
       - A trade is a "win" if profit_ratio > 0 (positive after fees)
       - A trade is a "loss" if profit_ratio <= 0
       - win_rate = (profitable_count / total_opportunities) * 100, as Decimal
       - avg_profit = mean of all profit_pct values, as Decimal
       - max_profit = max profit_pct, max_loss = min profit_pct
       - profit_distribution: bucket profit_pct values into ranges: "<0", "0.0-0.5", "0.5-1.0", "1.0-2.0", "2.0+"
       - All math uses decimal.Decimal — parse profit_pct and profit_ratio from string via Decimal(str(value))
       - Handle empty trades: return all-zero BacktestReport

    4. `format_report(report: BacktestReport) -> str`:
       - Returns a human-readable multiline string for stdout
       - Includes all metrics and profit distribution

    5. `save_report_json(report: BacktestReport, path: str) -> None`:
       - Writes BacktestReport as JSON to the given path
       - Uses json.dumps with default=str for Decimal serialization

    Follow Phase 1 patterns: docstrings with requirement IDs, logging module, Decimal everywhere.

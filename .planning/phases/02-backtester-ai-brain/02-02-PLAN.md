## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | I (Info Disclosure) | src/ai_brain.py | mitigate | ANTHROPIC_KEY loaded from .env only (never hardcoded), sent only over HTTPS via anthropic SDK's default transport. Key never logged — logger.debug messages reference "AI brain" not the key value |
| T-02-05 | T (Tampering) | src/ai_brain.py:_parse_response | mitigate | Parse Claude response with json.loads in try/except — malformed or malicious JSON returns None, never crashes bot. Only extract expected string fields (suggestion, new_threshold, reasoning) — ignore unexpected fields |
| T-02-06 | D (DoS) | src/ai_brain.py:review_trade | mitigate | Exponential backoff (1s, 2s, 4s) with 3 max retries prevents retry storms. asyncio.create_task ensures API latency never blocks main loop. Total worst-case delay: ~7s in background task only |
| T-02-07 | E (Elevation) | src/ai_brain.py | mitigate | AI suggestions are observe-only — logged to ai_reviews.jsonl but NEVER auto-applied to thresholds. No code path exists to modify PROFIT_THRESHOLD from AI output |
| T-02-08 | S (Spoofing) | src/config.py:ANTHROPIC_KEY | accept | Key loaded from .env file with standard file permissions. If attacker has .env read access, they already have XRPL_SECRET — ANTHROPIC_KEY is lower-value target |

**Part B — Create .streamlit/config.toml:**

Create the `.streamlit/` directory and `config.toml` file. Read `STREAMLIT_PORT` env var at runtime; if absent, default to 8501.

The config.toml content is static — port override via env var works differently with Streamlit: the `--server.port` CLI flag or `STREAMLIT_SERVER_PORT` env var overrides config.toml at runtime. So config.toml should just set the default:

```toml
[theme]
base = "dark"

[server]
port = 8501
headless = true
```

Add a comment in the file:
```toml
# Override port at runtime: STREAMLIT_SERVER_PORT=8502 streamlit run src/dashboard.py
# Or via CLI: streamlit run src/dashboard.py --server.port 8502
```

Note: The `STREAMLIT_PORT` env var from D-7 decision maps to Streamlit's own `STREAMLIT_SERVER_PORT` env var — document this in the comment.

`headless = true` suppresses the "want to contribute?" prompt on first run, appropriate for VPS deployment.
